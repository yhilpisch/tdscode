# Primer Notebooks

The Data Scientist — Book 0 · Python Primer for Data Science  
(c) Dr. Yves J. Hilpisch  
AI-Powered by GPT 5.x

These notebooks mirror the primer chapter arc and should read like compact,
self-contained learning sessions. Each notebook should include orienting
Markdown, runnable code, visible outputs, and short reflection prompts.

## Chapter coverage

- `01_colab_quickstart.ipynb` — notebook basics, runtime checks, first artifact,
  and the read → run → reflect → save loop.
- `02_notebook_habits.ipynb` — reproducible notebook habits, restart-and-run-all
  thinking, and lightweight session artifacts.
- `03_local_setup_and_portfolio.ipynb` — local environment reasoning, relative
  paths, portfolio structure, and README/journal scaffolding.
- `04_first_mini_project.ipynb` — a complete primer artifact that combines
  Markdown, code, a saved output, and a final reflection.

## Notebook style expectations

- Keep paths relative so notebooks run locally and in Colab.
- Explain each meaningful code block before or after it.
- Save at least one small artifact per notebook.
- End with a short reflection or “next step” note.
